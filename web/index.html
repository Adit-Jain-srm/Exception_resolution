Concurrency Model

  • Cases processed in parallel (Kafka partitions)
  • Investigators fan-out concurrently per case
  • LLM calls rate-limited via semaphore (max 20)
  • No shared mutable state between cases

LLM Strategy

  • Decision Engine: GPT-4o (deep reasoning)
  • Orchestrator: GPT-4o-mini (fast routing)
  • Investigators: No LLM (deterministic)
  • Failover: OpenAI → Anthropic → Rule-only

Safety Guarantees

  • LLMs advise, deterministic code executes
  • Action allowlist (no hallucinated actions)
  • Confidence floor enforcement
  • High-value gate (>$50K requires approval)
