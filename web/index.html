LLM Safety & Guardrail Architecture

Defense-in-depth framework ensuring no hallucinated, unsafe, or unauthorized action reaches production financial systems

Core Safety Principle

LLMs ADVISE. Deterministic code EXECUTES. No LLM output directly triggers a financial action. Every recommendation passes through schema validation, action allowlist, confidence thresholds, and idempotency checks before any side-effect occurs.

5Validation Layers
0Direct LLM Executions
8Allowed Actions Only

Pre-Inference Controls

Sanitize, constrain, and protect data before it reaches the LLM

LAYER 1 — INPUT
🔒
PII Data Isolation

Account numbers, beneficiary names, and personal identifiers masked with tokenization. Amounts and routing codes preserved (required for decisions). LLM never sees recoverable PII.

🛡
Prompt Injection Defense

All external-origin text (error messages, compliance notes, bank responses) sanitized through multi-pass filter. Control characters stripped. Layered prompt architecture isolates system instructions from user data.

✅
Schema Enforcement

Evidence bundle structure validated against Pydantic schema before prompt construction. Malformed or unexpected fields rejected — prevents context poisoning from corrupted upstream data.

📈
Token Budget & Context Control

Hard cap: 4,096 tokens input, 1,024 tokens output. Evidence truncated to decision-relevant fields only. Raw database rows never sent. Controls cost and prevents attention dilution.

Post-Inference Validation

Verify LLM output is safe, valid, and authorized before any action proceeds

LAYER 2 — OUTPUT
⛔
Action Allowlist Gate

LLM output MUST match one of exactly 8 predefined actions (AUTO_RETRY, REPAIR_AND_RETRY, HOLD_PENDING_FUNDS, HOLD_PENDING_INPUT, CANCEL_SAFELY, ESCALATE_COMPLIANCE, ESCALATE_OPERATIONS, DEFER_TO_NEXT_CYCLE). Any hallucinated action is immediately rejected.

📊
Confidence Floor Enforcement

Risk-tiered minimum confidence: LOW risk requires MEDIUM confidence, MEDIUM risk requires MEDIUM, HIGH risk requires HIGH. Decisions below floor auto-escalate to human — system never acts on low-confidence recommendations.

👁
Hallucination Detection

Cross-references all evidence IDs cited in the decision against the actual evidence bundle. If the LLM references evidence that doesn't exist, the response is fabricated — reject immediately and escalate.

💰
Financial Threshold Gate

Transactions exceeding configurable threshold (default: ₹50,000 / $50,000) require mandatory human approval regardless of confidence. Prevents high-value automated actions without oversight.

⚖
Rule-LLM Contradiction Detection

When the deterministic rule engine and LLM produce different recommendations, neither is trusted blindly. Case escalated to human review with both analyses presented. Protects against both rule gaps and LLM errors.

📜
Justification Integrity Check

Scans justification text for uncertainty markers ("I think", "probably", "might be"). Decisions must be stated factually based on evidence. Speculative language indicates the model is uncertain — reject and escalate.

Runtime Safety Controls

Operational guardrails protecting system availability, cost, and auditability

LAYER 3 — OPERATIONAL
🔄
Multi-Provider Failover

Primary: OpenAI GPT-4o → Failover: Anthropic Claude Sonnet → Degradation: Rule-only mode (no LLM). System never halts — gracefully degrades to deterministic processing. Human review flagged for degraded decisions.

⏱
Concurrency & Rate Control

Semaphore limits concurrent LLM calls (max 20). Priority queue ensures CRITICAL cases get inference slots first. Token bucket rate limiter per provider prevents quota exhaustion. Overflow cases degrade to rule-only path.

💲
Cost Circuit Breaker

Per-case cap: $0.05 (max 5 LLM calls). Daily budget alert at 80%. Monthly hard stop at 100% — all cases route to rule-only mode. Prevents runaway costs from retry loops or prompt engineering attacks.

📌
Model Version Pinning & Audit

Exact model version (e.g., gpt-4o-2024-11-20) recorded with every decision. Enables deterministic replay for regulatory audit. Version drift detected and alerted. Reproducibility gap logged if pinned version deprecated.

🛑
Kill Switch Matrix

Global disable (all auto-resolution stops). Per-rail disable (e.g., pause UPI during outage). Per-exception-type disable. Amount ceiling adjustment. All togglable in <60 seconds without deployment.
