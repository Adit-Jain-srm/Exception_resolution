Concurrency Model

  • Cases processed in parallel (Kafka partitions by payment_id)
  • Investigators fan-out concurrently per case (asyncio)
  • LLM calls rate-limited via semaphore (max 20 concurrent)
  • No shared mutable state between cases
🤖

LLM Strategy

  • Decision Engine: GPT-4o / Claude Sonnet
  • Orchestrator: GPT-4o-mini / Haiku
  • Investigators: No LLM (deterministic)
  • Failover: OpenAI → Anthropic → Rules-only
🛡

Safety Guarantees

  • LLMs advise, deterministic code executes
  • Action allowlist rejects hallucinated actions
  • Confidence floor auto-escalates to human
  • High-value gate: >₹50K needs approval

Explainability Guarantees

🔍

Evidence-Backed

  • Every claim references actual evidence from investigators
  • Numbers in justification come from source systems, not LLM generation
  • Guardrails reject justifications citing non-existent evidence
🔗

Rule-Traceable

  • Each explanation cites the exact rule ID that triggered
  • Rules are versioned and auditable in configuration
  • Override explanations preserve original + override reasoning
🔁

Reproducible

  • Same evidence bundle always generates same explanation
  • Temperature 0 + model pinning ensures determinism
  • Can be replayed for audit at any future date

LLM Guardrails Architecture

Multi-layer safety system preventing hallucinated or unsafe actions from reaching production systems

Layer 1: Input Guardrails

Before LLM
🔒
PII Masking

Account numbers and names masked before LLM sees data

✅
Schema Validation

Evidence structure validated before prompt construction

🛡
Injection Defense

External text sanitized against prompt injection attacks

📈
Token Budget

Hard cap: 4K input, 1K output per LLM call

Layer 2: Output Guardrails

After LLM
⛔
Action Allowlist

Only 8 predefined actions permitted. All others rejected.

📊
Confidence Floor

Low confidence decisions auto-escalate to human review

👁
Hallucination Detection

Rejects decisions referencing non-existent evidence IDs

💰
Amount Threshold

Transactions >₹50K require human approval

⚖
Contradiction Check

LLM vs rule-engine disagreement triggers escalation

Layer 3: Operational Guardrails

Runtime
🔄
Provider Failover

OpenAI → Anthropic → rule-only degradation

⏱
Rate Limiter

Priority queue: critical cases get LLM slots first

💲
Cost Circuit Breaker

Monthly budget cap with auto-disable at 100%

📌
Model Version Pinning

Exact version recorded for audit reproducibility
